FHIR Questionnaire Generator

Calls the validator's /itb/questionnaire service (GITB) to build a Questionnaire from a StructureDefinition profile, and renders it.
The root of the running validator HTTP service.
Must be loaded in the validator's context (load its IG first if it's not base FHIR).
Each is evaluated per ElementDefinition; an element is kept if ANY matches. Empty = whole profile.
The generator can't resolve terminology on its own — coded items come out bare. With this on, the page also calls /itb/package (expandValueSets=true), then uses the packaged profile's element bindings + expanded ValueSets to give every coded item real answer options, inlined as contained resources. The result is a self-contained, offline-renderable Questionnaire — the canonical use of the CRMI $package operation.
If blank, a few common versions are probed and the first that exists is used. Override only if auto-detect picks a version you don't want. Browse clinicaltables.nlm.nih.gov/lforms-versions/ for the full list.
